# Portfolio & Career Impact Summary — v1.1.0

*Domain-Specific RAG Evaluation & MLOps Platform*

## Resume / Portfolio Highlight Block

**Domain-Specific RAG Evaluation & MLOps Platform** | Personal Open-Source Project
`Python · TypeScript/React · NetworkX · RAGAS · Docker Compose · FastAPI · SQLite · MinIO · Helm`

- **Architected** a novel **Graph Context Relevance (GCR)** metric — the first offline,
  deterministic graph-topological evaluator for RAG context quality — combining Entity
  Overlap ($S_e$), Structural Connectivity ($S_c$), and Hub Noise Penalty ($P_h$) into a
  single O(N + E) composite score that detects retrieval fragmentation and hub-node dilution
  invisible to standard cosine-similarity-based tools.

- **Engineered** a production-grade 11-microservice Docker Compose platform (ports 8001–8009
  + MinIO + Insights Portal at 5173) with a SHA-256-addressed SQLite GraphStore, async
  RAGAS testset generator, statistical Welch Z-test drift monitor with Slack alerting, and
  LLM-powered Auto-Insights engine compatible with GPT-4o-mini, Ollama, vLLM, and Azure
  OpenAI — all wired through a single `EvaluationDispatcher` facade.

- **Delivered** a **733-test parallel test suite** (pytest-xdist · 369 eval-pipeline +
  364 services tests) with deterministic offline fallbacks, a two-stage `Dockerfile.test`
  matching CI environments exactly, and a `docker-compose.tools.yml` providing 15 Docker-
  native developer CLI tools — achieving 100% host-dependency-free operations across the
  entire development and CI lifecycle.

- **Implemented** a full-stack Insights Portal (React + TypeScript, Cytoscape.js KG
  visualiser, interactive QA Debugger with inline entity highlighting, multi-run analytics
  with p50/p90 baseline comparisons, PDF export, RTL/a11y compliance, and i18n) alongside
  an OWASP-hardened backend with path-traversal guards, scoped proxy ARGs to prevent
  secret leakage, and API key validation with actionable 503 responses.

- **Optimised** the full pipeline from raw CSV ingestion to evaluated KPIs and executive
  HTML/XLSX/PDF reports across 226 commits, 724 files, and 139,971 lines of intentional
  production code — eliminating ~200 legacy files and establishing a clean, fully
  purposeful codebase with `mypy`, `pytest`, and Docker-based CI as permanent quality gates.
